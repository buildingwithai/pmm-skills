## Launch Day Troubleshooting

### If Site Goes Down

- [ ] Product Monitor immediately notifies Commander
- [ ] Engineering team activates
- [ ] Communicate to users (Twitter, email if possible)
- [ ] Estimate time to fix
- [ ] Consider rollback if critical
- [ ] Post-mortem after resolution

**Communication template:**

"We're experiencing technical issues. Our team is on it. Updates in 15 minutes."

---

### If Payment Processing Fails

- [ ] Pause all paid acquisition immediately
- [ ] Put banner on site ("Free trial for now, payment issue being resolved")
- [ ] Fix payment provider issue
- [ ] Communicate to affected users
- [ ] Compensate affected users (extend trial, discount)

---

### If Negative Sentiment

- [ ] Identify source and severity
- [ ] Commander decides response approach
- [ ] Respond quickly, authentically
- [ ] Address concerns directly
- [ ] Don't delete negative comments (makes it worse)
- [ ] Learn from feedback

**Example:**

"You're right, [issue] is a problem. Here's how we're fixing it: [action]. Thanks for the honest feedback."

---

### If Low Engagement

- [ ] Don't panic (normal for first few hours)
- [ ] Review messaging (is value prop clear?)
- [ ] Check traffic sources (are we reaching right audience?)
- [ ] Test different headlines/copy
- [ ] Reach out to warm contacts directly
- [ ] Adjust expectations and iterate
